JS behind the scenes

JavaScript Engine

JavaScript Engine takes the code that we write to execute it.

What happens inside the engine?

Step 1: The code is parsed by a parser, which checks for the syntax of the code If the syntax is incorrect, it gives an error and stops the execution.

Step 2: If the syntax is correct, then the parser produces a data structure which is called Abstract syntax tree, which is translated into machine code Machine code can be executed by computer's processor only

Step 3: This machine code runs and does the work

Execution Contexts

The environment in which the JS code is executed. It is like a box or a container which stores variables and in which a piece of our code is evaluated and executed Execution context can also be said to be an object.

Execution Context Object containes three properties
  • Variable Object (VO) : This contains function arguments, variable declarations and function declarations
  • Scope chain: contains current variable objects, as well as the variable objects of all its parents
  • 'this' variable

Creation of execution context

The three properties of the execution context object are first created in the creation phase

Then comes the execution phase

In the execution phase, the code that created the current execution context is ran line by line

Properties of execution context in details


Variable Object (VO)

Creation
  • Argument object is created. This contains all the arguments that were passed into the function.
  • Code scans for function declarations. A property is then created inside the VO for each function pointing to that function.
  • Code scans for variable declarations. A property is created in VO and set to undefined
The last two points are combinedly what is called Hoisting In JS, functions and variables are hoisted, meaning that they are available before the execution starts.

IMP: Functions and Variables are hoisted in a different way. Functions are already defined before the execution phase, while variables are set to undefined.

Execution
  • Argument object is created. This contains all the arguments that were passed into the function.
  • Code scans for function declarations. A property is then created inside the VO for each function pointing to that function.
  • Code scans for variable declarations. A property is created in VO and set to undefined
The last two points are combinedly what is called Hoisting
